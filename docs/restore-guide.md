# คู่มือ Restore

> **คำเตือน:** การ Restore จะ**เขียนทับข้อมูลปัจจุบันทั้งหมด**ในฐานข้อมูลและไฟล์แนบ ด้วยข้อมูลจากไฟล์ Backup ที่เลือก การกระทำนี้ย้อนกลับไม่ได้ — หากไม่แน่ใจ ให้สำรองข้อมูลปัจจุบันไว้ก่อนเสมอ (ดู [คู่มือ Backup](backup-guide.md))

## วิธีที่ 1: Restore ผ่านหน้าเว็บ (แนะนำสำหรับการกู้คืนบนเครื่องเดิม)

1. เข้าสู่ระบบด้วยบัญชี Administrator
2. ไปที่เมนู **Backup / Restore**
3. เลือกไฟล์ Backup ที่ต้องการจากรายการ แล้วกด **"Restore"**
4. ระบบจะแสดงข้อความยืนยันเนื่องจากเป็นการกระทำที่ย้อนกลับไม่ได้ — อ่านให้แน่ใจก่อนกดยืนยัน
5. หลัง Restore สำเร็จ ให้ Restart แอปพลิเคชันเพื่อให้แน่ใจว่าทุก Session เชื่อมต่อกับข้อมูลใหม่:
   ```bash
   docker compose restart app
   ```

## วิธีที่ 2: Restore ด้วยคำสั่งจาก Terminal (ใช้เมื่อย้ายไปเครื่องใหม่)

ใช้วิธีนี้เมื่อติดตั้งระบบบนเครื่อง/เซิร์ฟเวอร์ใหม่ และต้องการนำข้อมูลจากเครื่องเดิมมาใช้:

1. ติดตั้งระบบบนเครื่องใหม่ตามคู่มือติดตั้ง ([Windows](install-windows.md) / [Linux](install-linux.md)) จนถึงขั้นตอน `docker compose up -d db` (เริ่มเฉพาะฐานข้อมูลเปล่า ยังไม่ต้องรัน `app`)

2. คัดลอกไฟล์ Backup (`.tar.gz`) จากเครื่องเดิมไปยังเครื่องใหม่ เช่นวางไว้ที่ `./restore-files/acy_backup_20260101_020000.tar.gz`

3. คัดลอกไฟล์เข้าไปใน Volume ของ Container:
   ```bash
   docker compose up -d app
   docker compose cp ./restore-files/acy_backup_20260101_020000.tar.gz app:/app/storage/backups/
   ```

4. รันคำสั่ง Restore ภายใน Container:
   ```bash
   docker compose exec app sh scripts/restore.sh /app/storage/backups/acy_backup_20260101_020000.tar.gz
   ```

5. Restart แอปพลิเคชัน:
   ```bash
   docker compose restart app
   ```

6. เข้าสู่ระบบด้วยบัญชีเดิมจากเครื่องต้นทาง เพื่อยืนยันว่าข้อมูล (ลูกค้า, สินค้า, ใบเสนอราคา, ผู้ใช้) ครบถ้วน

## สิ่งที่ Restore กลับคืนมา

- ฐานข้อมูลทั้งหมด (ผู้ใช้, สิทธิ์, ลูกค้า, สินค้า, ใบเสนอราคาทุก Revision, เอกสารบัญชีทุกประเภท, การชำระเงิน, Audit Log, การตั้งค่าทั้งหมด)
- ไฟล์แนบทั้งหมด (รูปสินค้า, โลโก้บริษัท, ลายเซ็น, ตราประทับ, Datasheet)

## สิ่งที่ต้องตั้งค่าใหม่หลัง Restore บนเครื่องใหม่ (ถ้าย้ายเครื่อง)

- ไฟล์ `.env` (รหัสผ่านฐานข้อมูล, `NEXTAUTH_SECRET`, `NEXTAUTH_URL` ให้ตรงกับ IP/Domain ของเครื่องใหม่) — ค่าเหล่านี้อยู่นอกฐานข้อมูล จึงไม่ได้ถูก Restore มาด้วยและต้องตั้งค่าใหม่ทุกครั้งที่ย้ายเครื่อง
- Firewall / Port Forwarding ตามคู่มือติดตั้ง

## แก้ปัญหา Restore ล้มเหลว

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ |
|---|---|---|
| `dropdb: database does not exist` | ปกติ ไม่ใช่ Error จริง (สคริปต์พยายามลบฐานข้อมูลเดิมก่อนเสมอ) | ไม่ต้องทำอะไร สคริปต์จะทำงานต่อได้ปกติ |
| `psql: connection refused` | Container `db` ยังไม่พร้อม | รอสักครู่แล้วลองใหม่ หรือตรวจสอบ `docker compose ps db` |
| ไฟล์ Backup เสียหาย / แตกไฟล์ไม่ได้ | ไฟล์ถูกคัดลอกไม่สมบูรณ์ | คัดลอกไฟล์ใหม่ ตรวจสอบขนาดไฟล์ตรงกับต้นทาง |
| หลัง Restore แล้ว Login ไม่ได้ | Session/Cookie เดิมค้างอยู่ | ล้าง Cookie ของเบราว์เซอร์ หรือเปิด Browser ใหม่แบบ Incognito |
